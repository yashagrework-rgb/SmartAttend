Project Overview

SmartAttend is a client-side Single Page Application (SPA) designed to track, analyze, and manage student attendance registries locally.

  • Zero External Dependencies: Built entirely with vanilla HTML5, CSS3, and ES6+ JS. No CDNs or frameworks.
  • Offline-First Architecture: Stores and processes data inside browser localStorage.
  • Departmental Target: Built specifically for coordinators to manage multiple classes independently.
  • Fast Load Speeds: Boots in under 85ms and consumes less than 1.2 MB of client memory.
Problem Statement

Traditional Registers

Pen-and-paper ledgers degrade physically over time, are vulnerable to tampering, and require hours of manual calculations to compute student attendance averages.

Centralized Cloud ERPs

Heavy subscription packages with high licensing fees. They are completely dependent on server networks; if the internet breaks down, class registers are blocked.

Project Objectives
  1. Create a serverless, local-first tool that runs offline on any desktop or mobile device.
  2. Build an **interactive dashboard** showing real-time attendance trends using custom SVGs.
  3. Support **class-wise registration** (filtering by Course & Semester) preventing database conflicts.
  4. Incorporate **CSV roster uploading** to eliminate manual student profiles key-in steps.
  5. Automate **parent warning alerts** by filtering defaulter students with less than 75% attendance.
System Architecture

SmartAttend runs a model-view-controller lifecycle computed entirely inside browser RAM, synchronizing updates to disk storage automatically:

[Browser DOM / User Actions]
    ⬇️ (Event Listeners & dynamic Hash Routing)
[Memory AppState Controller (script.js)] 🔄 [In-Memory Arrays]
    ⬇️ (JSON Stringify Serialization)
[Browser localStorage Engine] (Key-Value Store)
Local Database Schema

Students Array (`sams_students`)

Stores serialized list of student details.

[
  {
    "rollNumber": "BCA-2026-001",
    "fullName": "Aarav Sharma",
    "course": "BCA",
    "semester": "VI",
    "email": "aarav@college.edu"
  }
]

Logs Object (`sams_attendance`)

Maps date keys to student rolls and marked state.

{
  "2026-06-08": {
    "BCA-2026-001": "Present",
    "BCA-2026-002": "Absent"
  }
}
Core Modules & Features
  • Interactive Statistics: Displays present count, absent count, and aggregate percentages with pulsing pending CTA labels.
  • SVG Visual Polyline Chart: Dynamic coordinate rendering showing attendance trend calculations.
  • CSV Importer API: Standard browser FileReader streams that parse rosters, skip duplicate roll keys, and add students in bulk.
  • Non-Destructive Merging: Instructors mark classes separately without wiping logs taken by other coordinators on the same day.
Parent Warning Notices

The Analytics reports module isolates "defaulter" students with less than 75% attendance averages.

  • Automatically compiles metrics.
  • Calculates exact lectures missed.
  • Generates parent-ready letters.
  • Includes A4 media print CSS rules.

Conclusion & Future Scope

SmartAttend delivers an efficient, secure, and user-friendly solution for class-wise tracking. Its zero-cost operation and fast load times make it a viable alternative for department coordinators.

Future Enhancements:

  1. Integrate biometric check-ins (face recognition API inputs).
  2. Connect **SMS Twilio gateways** to alert parents automatically.
  3. Develop a secure **central REST server sync** for multi-teacher access.
